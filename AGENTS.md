# Repository Guidelines

## 项目结构与模块组织
- 章节内容位于根目录下以 `chNN-` 开头的文件夹（对应每一章）。
- 每个章节目录通常包含：
  - 该章的 `README.md` 笔记。
  - C 源码，如 `echo_server.c`、`hello_client.c`。
  - `bin/` 用于已编译的可执行文件。
  - 可选的资源文件（图片、样例数据等）。
- 仓库根目录包含总览 `README.md` 及相关文档。

## 构建、测试与开发命令
本仓库没有统一构建系统，按章节在对应目录内编译。

```sh
# 编译单个程序
mkdir -p bin
gcc -Wall -Wextra -O2 -o bin/hello_server hello_server.c

# 运行（按程序需要调整参数）
./bin/hello_server 8080
```

客户端/服务端成对示例需先启动服务端，再运行客户端：

```sh
./bin/hello_client 127.0.0.1 8080
```

## 编码风格与命名约定
- 语言：C。
- 缩进：4 个空格；大括号换行（Allman 风格）。
- 命名：函数和变量使用 `snake_case`；文件名描述角色（如 `echo_server.c`）。
- 可执行文件放入 `bin/`；源码保留在章节目录根部。
- 未配置格式化或 lint 工具；改动需与现有风格一致。

## 测试指南
- 未包含自动化测试框架。
- 通过编译并运行程序验证示例行为（如输出内容、连接流程）。
- 若新增脚本/测试，请在对应章节的 `README.md` 里说明运行方式。

## 提交与 PR 规范
- 近期提交多为简短摘要，偶尔使用 Conventional Commit，如 `docs(README): ...` 或 `chore(ide): ...`。
- 建议使用 `type(scope): summary`，消息简短明确。
- PR 应包含：
  - 变更内容与影响章节。
  - 构建/运行示例的方式。
  - 如有问题单请关联；仅在更新文档渲染或图片时提供截图。

## 贡献者注意事项
- 除非必要，避免提交体积较大的生成二进制文件。
- 章节笔记应保持简洁，并与对应代码示例一致。
- 本仓库主要用于 TCP/IP 网络编程学习。
- 始终用中文与我进行交流。
